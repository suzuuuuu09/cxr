# cx

`cx` は、YAML テンプレートからディレクトリ構成やファイルを生成する Rust 製 CLI です。

[English](./README.md)

## できること

- 利用可能なテンプレートを一覧表示する
- テンプレートのひな形を作成する
- テンプレート内の変数を置き換えてファイル群を生成する

## ビルド

```bash
cargo build
```

## 使い方

```bash
cargo run -- list
cargo run -- new <name>
cargo run -- <template> [-v VAR=VALUE ...] [-o OUTPUT]
```

インストール済みなら `cargo run --` の代わりに `cx` を使えます。

### コマンド

- `list` : 利用可能なテンプレートを表示
- `new <name>` : `~/.config/cx/` または `$XDG_CONFIG_HOME/cx/` に `<name>.yaml` を作成
- `<template>` : 指定したテンプレートからファイルを生成

## テンプレートの置き場所

テンプレートは次のいずれかに保存されます。

- `$XDG_CONFIG_HOME/cx/`
- `~/.config/cx/`

ファイル名は `<template>.yaml` です。

## テンプレート形式

テンプレートは YAML で定義します。

```yaml
name: "default template"
description: "A default template generated by cx"
variables:
  - project_name
  - author: "John Doe"
items:
  - type: "directory"
    name: "src"
    items:
      - type: "file"
        name: "main.py"
        content: |
          def main():
              print("Hello, {{ project_name }}!")
```

### 変数

- `- project_name` のように値なしで定義すると、生成時に入力を求められます
- `- author: "John Doe"` のように書くと、デフォルト値付きで入力できます
- `{{ project_name }}` のようなプレースホルダは、ファイル名・ディレクトリ名・ファイル内容で置換されます

### アイテム

- `type: directory` : ディレクトリを作成
- `type: file` : ファイルを作成

## 既存ファイルの扱い

同名のディレクトリやファイルがある場合は、上書きするかどうかを確認します。
