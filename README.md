# cx

`cx` is a Rust CLI that generates directory structures and files from YAML templates.

[日本語版](./README-ja.md)

## What it does

- Lists available templates
- Creates a starter template file
- Generates files by replacing variables in a template

## Build

```bash
cargo build
```

## Usage

```bash
cargo run -- list
cargo run -- new <name>
cargo run -- remove <name>
cargo run -- fzf
cargo run -- <template> [-v VAR=VALUE ...] [-o OUTPUT] [--dry-run] [--force|--backup|--skip]
```

If installed, you can use `cx` instead of `cargo run --`.

### Commands

- `list`: show available templates
- `new <name>`: create `<name>.yaml` under `~/.config/cx/` or `$XDG_CONFIG_HOME/cx/`
- `remove <name>` (`delete <name>`): remove an existing template file
- `fzf`: select a template via fzf and generate
- `<template>`: generate files from the specified template

## Template location

Templates are stored in one of these locations:

- `$XDG_CONFIG_HOME/cx/`
- `~/.config/cx/`

Each template file must be named `<template>.yaml`.

## Template format

Templates are defined in YAML.

```yaml
name: "default template"
description: "A default template generated by cx"
variables:
  - project_name
  - author: "John Doe"
pre_hook: "echo pre {{ project_name }}"
post_hook: "echo post {{ project_name }}"
items:
  - type: "directory"
    name: "src"
    items:
      - type: "file"
        name: "main.py"
        content: |
          def main():
              print("Hello, {{ project_name }}!")
```

### Variables

- Use `- project_name` to prompt for a value during generation
- Use `- author: "John Doe"` to provide a default value
- Placeholders such as `{{ project_name }}` are replaced in file names, directory names, and file contents

### Items

- `type: directory`: create a directory
- `type: file`: create a file

## Existing files

If a file or directory already exists, cx asks before overwriting it.

## Overwrite strategies

- `--force`: overwrite existing files without prompting
- `--backup`: rename existing files to `.bak` before writing (overwrites existing `.bak`)
- `--skip`: skip existing files without prompting

Directories are reused; overwrite strategies apply to files only. `--backup` will rename existing directories to `.bak` before creating a new directory.

## Dry run

Use `--dry-run` (`-n`) to preview the generated items without writing files. It still checks existing paths and reports when it would overwrite them.

## Hooks

Use `pre_hook` and `post_hook` in the template YAML to run a command before and after generation. Hooks run in the output directory and support `{{ var }}` substitution. If a hook fails, generation stops.

```yaml
pre_hook: "echo pre {{ project_name }}"
post_hook: "echo post {{ project_name }}"
```

Hooks are executed with `/bin/sh -c`. Dry run does not execute hooks.

## fzf selection

`cx fzf` uses [fzf](https://github.com/junegunn/fzf) to select a template and then generates files. If fzf is not installed or the selection is cancelled, cx exits with an error.
